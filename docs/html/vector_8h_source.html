1 /*******************************************************************************
2 * Vectors
3 *
4 * The Vector and Matrix types here are used throughout the rest of the Robotics
5 * Cape library behind the scenes, but are also available to the user along with
6 * a collection of hardware-accelerated linear algebra functions. The premise
7 * of both types is that a small rc_vector_t and rc_matrix_t struct contains
8 * information about type's size and a pointer to where dynamically allocated
9 * memory exists that stores the actual data for the vector or matrix. Use
10 * rc_alloc_vector and rc_alloc_matrix to dynamically allocate memory for each
11 * new vector or matrix. Then use rc_free_vector and rc_free_matrix to free the
12 * memory when you are done using it. See the remaining vector, matrix, and
13 * linear algebra functions for more details.
14 *
15 * @ int rc_alloc_vector(rc_vector_t* v, int length)
16 *
17 * Allocates memory for vector v to have specified length. If v is initially the
18 * right length then nothing is done and the data in v is preserved. If v is
19 * uninitialized or of the wrong length then any existing memory is freed and new
20 * memory is allocated, helping to prevent accidental memory leaks. The contents
21 * of the new vector is not guaranteed to be anything in particular.
22 * Returns 0 if successful, otherwise returns -1. Will only be unsuccessful if
23 * length is invalid or there is insufficient memory available.
24 *
25 * @ int rc_free_vector(rc_vector_t* v)
26 *
27 * Frees the memory allocated for vector v and importantly sets the length and
28 * initialized flag of the rc_vector_t struct to 0 to indicate to other functions
29 * that v no longer points to allocated memory and cannot be used until more
30 * memory is allocated such as with rc_alloc_vector or rc_vector_zeros.
31 * Returns 0 on success. Will only fail and return -1 if it is passed a NULL
32 * pointer.
33 *
34 * @ rc_vector_t rc_empty_vector()
35 *
36 * Returns an rc_vector_t with no allocated memory and the initialized flag set
37 * to 0. This is useful for initializing vectors when they are declared since
38 * local variables declared in a function without global variable scope in C are
39 * not guaranteed to be zeroed out which can lead to bad memory pointers and
40 * segfaults if not handled carefully. We recommend initializing all
41 * vectors with this function before using rc_alloc_matrix or any other function.
42 *
43 * @ int rc_vector_zeros(rc_vector_t* v, int length)
44 *
45 * Resizes vector v and allocates memory for a vector with specified length.
46 * The new memory is pre-filled with zeros. Any existing memory allocated for v
47 * is freed if necessary to avoid memory leaks.
48 * Returns 0 on success or -1 on error.
49 *
50 * @ int rc_vector_ones(rc_vector_t* v, int length)
51 *
52 * Resizes vector v and allocates memory for a vector with specified length.
53 * The new memory is pre-filled with floating-point ones. Any existing memory
54 * allocated for v is freed if necessary to avoid memory leaks.
55 * Returns 0 on success or -1 on error.
56 *
57 * @ int rc_random_vector(rc_vector_t* v, int length)
58 *
59 * Resizes vector v and allocates memory for a vector with specified length.
60 * The new memory is pre-filled with random floating-point values between -1.0f
61 * and 1.0f. Any existing memory allocated for v is freed if necessary to avoid
62 * memory leaks.
63 * Returns 0 on success or -1 on error.
64 *
65 * @ int rc_vector_fibonnaci(rc_vector_t* v, int length)
66 *
67 * Creates a vector of specified length populated with the fibonnaci sequence.
68 * Returns 0 on success or -1 on error.
69 *
70 * @ int rc_vector_from_array(rc_vector_t* v, float* ptr, int length)
71 *
72 * Sometimes you will have a normal C-array of floats and wish to convert to
73 * rc_vector_t format for use with the other linear algebra functions.
74 * This function duplicates the contents of an array of floats into vector v and
75 * ensures v is sized correctly. Existing data in v (if any) is freed and lost.
76 * Returns 0 on success or -1 on failure.
77 *
78 * @ int rc_duplicate_vector(rc_vector_t a, rc_vector_t* b)
79 *
80 * Allocates memory for a duplicate of vector a and copies the contents into
81 * the new vector b. Simply making a copy of the rc_vector_t struct is not
82 * sufficient as the rc_vector_t struct simply contains a pointer to the memory
83 * allocated to contain the contents of the vector. rc_duplicate_vector sets b
84 * to be a new rc_vector_t with a pointer to freshly-allocated memory.
85 * Returns 0 on success or -1 on error.
86 *
87 * @ int rc_set_vector_entry(rc_vector_t* v, int pos, float val)
88 *
89 * Sets the entry of vector 'v' in position 'pos' to 'val' where the position is
90 * zero-indexed. In practice this is never used as it is much easier for the user
91 * to set values directly with this code:
92 *
93 * v.d[pos]=val;
94 *
95 * However, we provide this function for completeness. It is not strictly
96 * necessary for v to be provided as a pointer as a copy of the struct v
97 * would also contain the correct pointer to the original vector's allocated
98 * memory. However, in this library we use the convention of passing an
99 * rc_vector_t struct or rc_matrix_struct as a pointer when its data is to be
100 * modified by the function, and as a normal argument when it is only to be read
101 * by the function.
102 * Returns 0 on success or -1 on error.
103 *
104 * @ float rc_get_vector_entry(rc_vector_t v, int pos)
105 *
106 * Returns the entry of vector 'v' in position 'pos' where the position is
107 * zero-indexed. Returns -1.0f on failure and prints an error message to stderr.
108 * In practice this is never used as it is much easier for the user to read
109 * values directly with this code:
110 *
111 * val = v.d[pos];
112 *
113 * However, we provide this function for completeness. It also provides sanity
114 * checks to avoid possible segfaults.
115 *
116 * @ int rc_print_vector(rc_vector_t v)
117 *
118 * Prints to stdout the contents of vector v in one line. This is not advisable
119 * for extremely long vectors but serves for quickly debugging or printing
120 * results. It prints 4 decimal places with padding for a sign. We recommend
121 * rc_print_vector_sci() for very small or very large numbers where scientific
122 * notation would be more appropriate. Returns 0 on success or -1 on failure.
123 *
124 * @ int rc_print_vector_sci(rc_vector_t v)
125 *
126 * Prints to stdout the contents of vector v in one line. This is not advisable
127 * for extremely long vectors but serves for quickly debugging or printing
128 *
129 * @ int rc_vector_times_scalar(rc_vector_t* v, float s)
130 *
131 * Multiplies every entry in vector v by scalar s. It is not strictly
132 * necessary for v to be provided as a pointer since a copy of the struct v
133 * would also contain the correct pointer to the original vector's allocated
134 * memory. However, in this library we use the convention of passing an
135 * rc_vector_t struct or rc_matrix_struct as a pointer when its data is to be
136 * modified by the function, and as a normal argument when it is only to be read
137 * by the function.
138 * Returns 0 on success or -1 on failure.
139 *
140 * @ float rc_vector_norm(rc_vector_t v, float p)
141 *
142 * Just like the matlab norm(v,p) function, returns the vector norm defined by
143 * sum(abs(v)^p)^(1/p), where p is any positive real value. Most common norms
144 * are the 1 norm which gives the sum of absolute values of the vector and the
145 * 2-norm which is the square root of sum of squares.
146 * for infinity and -infinity norms see vector_max and vector_min
147 *
148 * @ int rc_vector_max(rc_vector_t v)
149 *
150 * Returns the index of the maximum value in v or -1 on failure. The value
151 * contained in the returned index is the equivalent to the infinity norm. If the
152 * max value occurs multiple times then the first instance is returned.
153 *
154 * @ int rc_vector_min(rc_vector_t v)
155 *
156 * Returns the index of the minimum value in v or -1 on failure. The value
157 * contained in the returned index is the equivalent to the minus-infinity norm.
158 * If the min value occurs multiple times then the first instance is returned.
159 *
160 * @ float rc_std_dev(rc_vector_t v)
161 *
162 * Returns the standard deviation of the values in a vector or -1.0f on failure.
163 *
164 * @ float rc_vector_mean(rc_vector_t v)
165 *
166 * Returns the mean (average) of all values in vector v or -1.0f on error.
167 *
168 * @ int rc_vector_projection(rc_vector_t v, rc_vector_t e, rc_vector_t* p)
169 *
170 * Populates vector p with the projection of vector v onto e.
171 * Returns 0 on success, otherwise -1.
172 *
173 * @ float rc_vector_dot_product(rc_vector_t v1, rc_vector_t v2)
174 *
175 * Returns the dot product of two equal-length vectors or floating-point -1.0f
176 * on error.
177 *
178 * @ int rc_vector_outer_product(rc_vector_t v1, rc_vector_t v2, rc_matrix_t* A)
179 *
180 * Computes v1 times v2 where v1 is a column vector and v2 is a row vector.
181 * Output is a matrix with same rows as v1 and same columns as v2.
182 * Returns 0 on success, otherwise -1.
183 *
184 * @ int rc_vector_cross_product(rc_vector_t v1, rc_vector_t v2, rc_vector_t* p)
185 *
186 * Computes the cross-product of two vectors, each of length 3. The result is
187 * placed in vector p and and existing memory used by p is freed and lost.
188 * Returns 0 on success, otherwise -1.
189 *
190 * @ int rc_vector_sum(rc_vector_t v1, rc_vector_t v2, rc_vector_t* s)
191 *
192 * Populates vector s with the sum of vectors v1 and v2. Any existing memory
193 * allocated for s is freed and lost, new memory is allocated if necessary.
194 * Returns 0 on success, otherwise -1.
195 *
196 * @ int rc_vector_sum_inplace(rc_vector_t* v1, rc_vector_t v2)
197 *
198 * Adds vector v2 to v1 and leaves the result in v1. The original contents of v1
199 * are lost and v2 is left untouched.
200 * Returns 0 on success, otherwise -1.
201 *******************************************************************************/
