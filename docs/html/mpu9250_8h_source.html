1 /******************************************************************************
2 * 9-AXIS IMU
3 *
4 * The Robotics Cape features an Invensense MPU9250 9-axis IMU. This API allows
5 * the user to configure this IMU in two modes: RANDOM and DMP
6 *
7 * RANDOM: The accelerometer, gyroscope, magnetometer, and thermometer can be
8 * read directly at any time. To use this mode, call rc_initialize_imu() with your
9 * imu_config and imu_data structs as arguments as defined below. You can then
10 * call rc_read_accel_data, rc_read_gyro_data, rc_read_mag_data, or rc_read_imu_temp
11 * at any time to get the latest sensor values.
12 *
13 * DMP: Stands for Digital Motion Processor which is a feature of the MPU9250.
14 * in this mode, the DMP will sample the sensors internally and fill a FIFO
15 * buffer with the data at a fixed rate. Furthermore, the DMP will also calculate
16 * a filtered orientation quaternion which is placed in the same buffer. When
17 * new data is ready in the buffer, the IMU sends an interrupt to the BeagleBone
18 * triggering the buffer read followed by the execution of a function of your
19 * choosing set with the rc_set_imu_interrupt_func() function.
20 *
21 * @ enum rc_accel_fsr_t rc_gyro_fsr_t
22 *
23 * The user may choose from 4 full scale ranges of the accelerometer and
24 * gyroscope. They have units of gravity (G) and degrees per second (DPS)
25 * The defaults values are A_FSR_4G and G_FSR_1000DPS respectively.
26 *
27 * enum rc_accel_dlpf_t rc_gyro_dlpf_t
28 *
29 * The user may choose from 7 digital low pass filter constants for the
30 * accelerometer and gyroscope. The filter runs at 1kz and helps to reduce sensor
31 * noise when sampling more slowly. The default values are ACCEL_DLPF_184
32 * GYRO_DLPF_250. Lower cut-off frequencies incur phase-loss in measurements.
33 *
34 * @ struct rc_imu_config_t
35 *
36 * Configuration struct passed to rc_initialize_imu and rc_initialize_imu_dmp. It is
37 * best to get the default config with rc_default_imu_config() function and
38 * modify from there.
39 *
40 * @ struct rc_imu_data_t
41 *
42 * This is the container for holding the sensor data from the IMU.
43 * The user is intended to make their own instance of this struct and pass
44 * its pointer to imu read functions.
45 *
46 * @ rc_imu_config_t rc_default_imu_config()
47 *
48 * Returns an rc_imu_config_t struct with default settings. Use this as a starting
49 * point and modify as you wish.
50 *
51 * @ int rc_initialize_imu(rc_imu_data_t *data, rc_imu_config_t conf)
52 *
53 * Sets up the IMU in random-read mode. First create an instance of the imu_data
54 * struct to point to as rc_initialize_imu will put useful data in it.
55 * rc_initialize_imu only reads from the config struct. After this, you may read
56 * sensor data.
57 *
58 * @ int rc_read_accel_data(rc_imu_data_t *data)
59 * @ int rc_read_gyro_data(rc_imu_data_t *data)
60 * @ int rc_read_mag_data(rc_imu_data_t *data)
61 * @ int rc_read_imu_temp(rc_imu_data_t* data)
62 *
63 * These are the functions for random sensor sampling at any time. Note that
64 * if you wish to read the magnetometer then it must be enabled in the
65 * configuration struct. Since the magnetometer requires additional setup and
66 * is slower to read, it is disabled by default.
67 *
68 ******************************************************************************/
